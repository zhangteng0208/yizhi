# 极空间 NAS 部署指南

## 一、准备工作

### 1.1 数据文件位置
- 本地数据：`/Users/mac/projects/yizhi/daizhigev20/` (6.9GB)
- 包含：医藏、诗藏、易藏、史藏、子藏、儒藏、佛藏等

### 1.2 需要上传到 NAS 的文件
```
nas-deploy/
├── docker-compose.yml    # Docker 配置
├── nginx.conf           # Nginx 配置
└── 部署指南.md          # 本文件

daizhigev20/             # 数据文件（6.9GB）
```

## 二、NAS 端操作

### 2.1 创建目录结构

在极空间 NAS 上创建以下目录：

```bash
# SSH 登录 NAS 后执行
mkdir -p /volume1/docker/yizhi-deploy
mkdir -p /volume1/docker/yizhi-data
```

### 2.2 上传文件

**方法 1：通过 SMB/NFS 共享**
1. 在 Mac 上挂载 NAS 共享目录
2. 复制文件：
   - `nas-deploy/` 下的文件 → `/volume1/docker/yizhi-deploy/`
   - `daizhigev20/` 整个目录 → `/volume1/docker/yizhi-data/classics/`

**方法 2：通过 SCP 命令**
```bash
# 在 Mac 上执行
cd /Users/mac/projects/yizhi

# 上传配置文件
scp nas-deploy/docker-compose.yml root@NAS_IP:/volume1/docker/yizhi-deploy/
scp nas-deploy/nginx.conf root@NAS_IP:/volume1/docker/yizhi-deploy/

# 上传数据文件（需要较长时间）
scp -r daizhigev20 root@NAS_IP:/volume1/docker/yizhi-data/classics/
```

### 2.3 启动 Docker 服务

SSH 登录 NAS，执行：

```bash
cd /volume1/docker/yizhi-deploy
docker-compose up -d
```

### 2.4 验证服务

```bash
# 查看容器状态
docker ps | grep yizhi

# 查看日志
docker logs yizhi-json-server

# 测试访问（在 NAS 上）
curl http://localhost:8080
```

## 三、公网访问配置

### 3.1 极空间内网穿透

1. 打开极空间 App
2. 进入「设置」→「网络」→「内网穿透」
3. 添加端口映射：
   - 内部端口：`8080`
   - 外部端口：`59000`（或其他可用端口）
   - 协议：`HTTP`

### 3.2 获取公网地址

内网穿透配置完成后，会得到类似：
```
https://your-nas-id.zspace.cn:59000
```

### 3.3 测试公网访问

```bash
# 在任意设备上测试
curl https://your-nas-id.zspace.cn:59000/classics/daizhigev20/医藏/xxx.txt
```

## 四、前端配置修改

修改前端代码，将数据文件 URL 指向 NAS：

## 八、注意事项

1. **CORS 已配置**：允许所有域名跨域访问
2. **只读挂载**：数据目录以只读方式挂载，防止误修改
3. **自动重启**：容器配置了 `restart: unless-stopped`
4. **端口冲突**：如果 8080 端口被占用，修改 `docker-compose.yml` 中的端口
5. **防火墙**：确保 NAS 防火墙允许 8080 端口访问
6. **HTTPS**：极空间内网穿透自动提供 HTTPS

## 九、故障排查

### 容器无法启动
```bash
docker logs yizhi-json-server
```

### 文件无法访问
```bash
# 检查文件权限
ls -la /volume1/docker/yizhi-data/classics/

# 进入容器检查
docker exec -it yizhi-json-server sh
ls -la /usr/share/nginx/html/
```

### 跨域问题
检查浏览器控制台，确认 CORS 头是否正确返回。

## 十、性能优化（可选）

如果访问量大，可以添加缓存配置：

```nginx
# 在 nginx.conf 中添加
location ~ \.(json|txt)$ {
    expires 1d;
    add_header Cache-Control "public, immutable";
}
```
