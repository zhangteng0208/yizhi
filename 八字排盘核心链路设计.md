> 版本: v1.0 | 日期: 2026-02-24
> 范围: 用户输入 → 八字排盘 → AI 解读 → 结果展示

## 一、整体流程

```
用户输入出生信息 → 前端提交 → 后端排盘计算 → 调用 AI 生成解读 → 存储结果 → 返回前端展示
```

## 二、八字排盘引擎

### 2.1 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | ✅ | 姓名 |
| gender | enum | ✅ | 男/女 |
| birthYear | number | ✅ | 出生年（公历） |
| birthMonth | number | ✅ | 出生月（公历） |
| birthDay | number | ✅ | 出生日（公历） |
| birthHour | string | ✅ | 时辰（子丑寅卯...） |
| isLunar | boolean | ❌ | 是否农历输入，默认 false |
| birthProvince | string | ❌ | 出生省份（用于真太阳时校正） |
| birthCity | string | ❌ | 出生城市 |

### 2.2 排盘计算步骤

#### Step 1: 公历转农历
- 使用 `lunar-javascript` npm 包（纯 JS，无外部依赖）
- 公历日期 → 农历年月日 + 闰月标识

#### Step 2: 干支推算
- **年柱**: 农历年 → 天干地支（六十甲子循环）
- **月柱**: 年干 + 农历月 → 月干支（五虎遁月法）
- **日柱**: 公历日期 → 日干支（查万年历或算法推算）
- **时柱**: 日干 + 时辰 → 时干支（五鼠遁时法）

#### Step 3: 五行统计
- 八个字（4天干 + 4地支）→ 统计金木水火土数量
- 地支藏干也纳入统计（本气、中气、余气）

#### Step 4: 日主强弱判断
- 日干为"我"（日主）
- 统计生我、助我 vs 克我、泄我、耗我的力量
- 综合月令、坐支、其他干支 → 判断身强/身弱

#### Step 5: 用神/忌神确定
- 身强 → 用神取克泄耗（官杀、食伤、财星）
- 身弱 → 用神取生助（印星、比劫）
- 特殊格局（从强、从弱、专旺等）单独处理

#### Step 6: 十神关系
- 以日干为基准，推算其余七个字的十神
- 正官、七杀、正印、偏印、比肩、劫财、食神、伤官、正财、偏财

#### Step 7: 大运排列
- 根据性别 + 年干阴阳 → 确定顺排/逆排
- 从月柱出发，每步大运 10 年
- 计算起运年龄

### 2.4 技术选型

| 方案 | 包名 | 说明 |
|------|------|------|
| 推荐 | `lunar-javascript` | 纯 JS，支持农历/干支/节气/八字，Star 1.5k+ |

## 三、AI 解读服务

### 3.1 架构设计

```
排盘数据 → Prompt 模板填充 → LLM API 调用 → 结构化解读 → 存储
```

### 3.2 Prompt 设计

采用"角色 + 数据 + 任务 + 格式"四段式 prompt：

```
你是一位精通八字命理的资深命理师，拥有 30 年实战经验。
请根据以下八字信息，为求测者提供专业、详细的命理分析。

【求测者信息】
姓名：{name}
性别：{gender}
出生：{birthInfo}（农历 {lunarInfo}）

【八字排盘】
年柱：{yearPillar}  月柱：{monthPillar}  日柱：{dayPillar}  时柱：{hourPillar}
日主：{dayMaster}（{strength}）
五行分布：金{metal} 木{wood} 水{water} 火{fire} 土{earth}
五行缺失：{missing}
用神：{useful}  忌神：{harmful}
格局：{pattern}

【请输出以下内容】
1. **命局总论**（200字）：整体命格特点和人生基调
2. **性格分析**（150字）：性格优势和需注意之处
3. **事业运势**（150字）：适合的行业方向和发展建议
4. **财运分析**（100字）：财运特点和理财建议
5. **感情婚姻**（100字）：感情模式和婚姻建议
6. **健康提醒**（80字）：需关注的健康方面
7. **流年运势**（当年，100字）：今年的运势重点
8. **改运建议**：幸运色、幸运数字、吉利方位、宜忌事项
9. **综合评分**：1-100 分

请用温和专业的语气，避免过于绝对的判断。输出 JSON 格式。
```

### 3.3 输出格式

```json
{
  "score": 85,
  "summary": "命局总论文本...",
  "sections": [
    { "title": "命局总论", "content": "..." },
    { "title": "性格分析", "content": "..." },
    { "title": "事业运势", "content": "..." },
    { "title": "财运分析", "content": "..." },
    { "title": "感情婚姻", "content": "..." },
    { "title": "健康提醒", "content": "..." },
    { "title": "流年运势", "content": "..." }
  ],
  "advice": {
    "luckyColor": "金色、白色",
    "luckyNumber": "4、9",
    "luckyDirection": "西方、西北方",
    "suitable": "求财、签约、出行",
    "avoid": "动土、诉讼"
  }
}
```

### 3.4 LLM 选型

| 优先级 | 模型 | 说明 |
|--------|------|------|
| 1 | DMXAPI 中转 | 已有 key，支持 OpenAI 兼容格式，成本低 |
| 2 | DeepSeek | 中文理解强，命理领域表现好，价格便宜 |
| 3 | OpenAI GPT-4o | 兜底方案，质量最稳定 |

通过 DMXAPI 统一调用，后端只需对接 OpenAI SDK 兼容接口。

### 3.5 性能优化

- **流式输出**: 使用 SSE（Server-Sent Events）实时推送解读内容到前端
- **缓存策略**: 相同八字 + 相同服务 → Redis 缓存 24h（不同人同八字，解读可复用）
- **超时处理**: LLM 调用超时 30s，失败后返回基础排盘数据 + "解读生成中"状态
- **异步模式**: 复杂解读可先返回排盘数据，AI 解读异步生成后推送

## 五、API 设计

### 5.1 提交测算

```
POST /api/divination
Authorization: Bearer {token}

Request:
{
  "serviceCode": "bazi",
  "profileId": "uuid",        // 可选，使用已有档案
  "name": "张三",
  "gender": "male",
  "birthYear": 1988,
  "birthMonth": 2,
  "birthDay": 8,
  "birthHour": "寅",
  "isLunar": false,
  "question": "想了解整体运势"  // 可选
}

Response:
{
  "code": 0,
  "data": {
    "recordId": "uuid",
    "status": "processing",
    "baziData": { ... },       // 排盘数据（立即返回）
    "result": null              // AI 解读（异步生成）
  }
}
```

### 5.2 获取结果（轮询或 SSE）

```
GET /api/divination/{id}

Response:
{
  "code": 0,
  "data": {
    "recordId": "uuid",
    "status": "completed",
    "baziData": { ... },
    "result": {
      "score": 85,
      "sections": [...],
      "advice": { ... }
    }
  }
}
```

### 5.3 SSE 流式推送（可选）

```
GET /api/divination/{id}/stream
Accept: text/event-stream

event: bazi
data: { "四柱": { ... } }

event: section
data: { "title": "命局总论", "content": "..." }

event: section
data: { "title": "性格分析", "content": "..." }

event: done
data: { "score": 85, "advice": { ... } }
```

## 六、数据存储

测算记录存入 `divination_records` 表：

| 字段 | 存储内容 |
|------|----------|
| input_params | 用户输入（姓名、出生信息等） |
| divination_data | 排盘原始数据（四柱、五行、大运等） |
| result | AI 解读结果（JSON） |
| result_summary | 解读摘要（用于列表展示） |
| ai_model | 使用的 AI 模型名称 |
| status | 0=处理中 1=完成 2=失败 |

## 八、风险与注意事项

1. **AI 输出质量**: 命理解读需要专业性，prompt 需反复调优
2. **LLM 成本**: 每次解读约 2000-3000 tokens，DMXAPI 成本约 ¥0.02-0.05/次
3. **合规风险**: 避免"算命"等敏感词，使用"命理分析""运势参考"等表述
4. **缓存命中**: 相同八字可复用解读，降低 API 调用成本
5. **lunar-javascript 精度**: 需验证 1900-2100 年范围内的排盘准确性
